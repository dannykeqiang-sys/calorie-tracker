# 深色模式文字可读性修复报告

## 背景
深色模式下大量文字看不清，根因是组件层硬编码了 `rgba(255,255,255,...)` / `#fff` 作为卡片/面板背景，以及 `rgba(0,0,0,...)` 作为文字色/边框/交互态。这些值在浅色模式正常，但在深色模式下不随主题切换，导致「白底 + 深字」在深色背景上失效或对比度极低。

修复思路：将所有「文字背后的背景/边框/交互态」硬编码值替换为 `global.css` 已定义的主题感知 CSS 变量，使其在浅色/深色模式下自动适配；对纯装饰性效果（阴影、图片/视频遮罩、模态遮罩、彩色横幅上的玻璃层与白字）予以保留。

## 使用的 CSS 变量
- 表面：`--ck-surface-subtle / --ck-surface-medium / --ck-surface-strong / --ck-surface-overlay`
- 边框：`--ck-border-subtle / --ck-border-medium / --ck-border-strong`
- 交互态：`--ck-hover / --ck-active`
- 图表 tooltip：`--ck-chart-tooltip-bg / --ck-chart-tooltip-fg`、网格 `--ck-chart-grid`
- Tailwind：`bg-card` / `bg-card/XX` / `border-card/XX`、`--card` / `--background`

## 映射规则
- 背景 `rgba(255,255,255,X)`：X≤0.5→`--ck-surface-subtle`；0.5<X≤0.75→`--ck-surface-medium`；0.75<X≤0.9→`--ck-surface-strong`；X>0.9→`--card`
- 边框 `rgba(0,0,0,X)`：X≤0.06→`--ck-border-subtle`；0.06<X≤0.12→`--ck-border-medium`；X>0.12→`--ck-border-strong`
- 交互态/浅色块背景 `rgba(0,0,0,X)`：X≤0.05→`--ck-hover`；X>0.05→`--ck-active`
- Tailwind：`bg-white[/XX]`→`bg-card[/XX]`，`border-white/XX`→`border-card/XX`
- SVG tooltip：底 `--ck-chart-tooltip-bg`，字 `--ck-chart-tooltip-fg`

## 按规则保留未改（合理）
- 阴影系统（boxShadow / textShadow / drop-shadow 中的 rgba）
- 模态遮罩层：`rgba(0,0,0,0.38~0.52)`（MealCardSlot、AIDrawer、WeightRecord* 等）
- 图片/视频上的文字遮罩渐变与白字：MealCarousel 底部标签渐变、卡片图片顶部压暗渐变、VideoIntro 视频遮罩、DesktopParallaxSlider 全屏图片上的文字与控件
- 玻璃高光边框/内阴影：卡片 `borderTop/Left` 的白色 1.5px 玻璃描边、inset 白色高光
- 彩色横幅/按钮上的玻璃层与白字：DesktopHeader 激活态、AnalyticsPanel / WeeklyStatsModal 横幅上的图标容器与关闭按钮、图表彩色柱/金字塔层上的白字、图例白色圆点
- 装饰性径向高光：AnalyticsPanel / WeeklyStatsModal 横幅 radial-gradient 光晕
- TutorialOverlay 引导卡：位于深色聚光灯遮罩之上，浅/深模式均为浅色卡 + 深字，两种模式下均可读，保留
- AppIcon inset 高光（影响极小，按清单保留）
- 次要滚动条 `scrollbarColor`（视觉影响极小）
