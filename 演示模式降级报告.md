# 演示模式降级报告

## 目标回顾
1. API Key 只存在 localStorage，禁止出现在任何源码文件（含 .env）中
2. AI 输入框示例文本改为完整一天（三餐+加餐+运动+饮水）的自然语言描述
3. 实现本地降级解析器：无 API Key 或 API 失败时，用关键词匹配 + 营养数据库智能解析并回填

## 改动清单

### 新增 `src/utils/demoFallback.ts`
本地食物营养数据库 + 关键词解析器，导出 `parseMixedMealsLocally(input)`，返回与 `parseMixedMeals` 完全一致的 `MixedMealResult` 格式。

- 内置 **65 种**常见食物营养数据（name/calories/protein/carbs/fat/sodium），覆盖早餐/午餐/晚餐/加餐/液体类
- 内置 **13 种**运动消耗数据（按 30 分钟基准，支持时长缩放）
- 解析逻辑：
  1. 按逗号/句号/分号/换行分段
  2. 每段识别时间关键词（早上→breakfast、中午→lunch、晚上→dinner、下午/加餐→snack），并在无时间线索时沿用上一段的餐段上下文
  3. 长关键词优先匹配并"消费"命中区域，避免子串重复计入（如 `糙米饭` 不会再命中 `米饭`）
  4. 识别中文/阿拉伯数量词（一碗/两个/三杯/半个等）按倍数换算营养
  5. 识别液体并按含水率估算含水量，写入 `water_logs`（有热量的液体同时进餐段）
  6. 识别运动，支持"30分钟""1小时"等时长并缩放消耗
  7. 生成温暖的 `analysis_summary`

