- **数据库**: SQLite (better-sqlite3)
- **认证**: JWT (jsonwebtoken)

### 数据库结构
```sql
-- 6张核心表
1. users           - 用户信息
2. user_profiles   - 用户详细资料
3. daily_records   - 每日饮食记录
4. data_backups    - 数据备份
5. invite_codes    - 邀请码
6. admin_config    - 管理员配置
```

### API端点

#### 认证API
- `POST /api/auth/login` - 用户登录/注册
- `POST /api/auth/admin` - 管理员登录

#### 用户API (需Token)
- `GET /api/user/profile` - 获取个人资料
- `PUT /api/user/profile` - 更新个人资料
- `GET /api/user/records/:date` - 获取指定日期记录
- `PUT /api/user/records/:date` - 保存记录
- `POST /api/user/backup` - 创建备份
- `GET /api/user/backups` - 获取备份列表
- `GET /api/user/backups/:id` - 获取备份详情

#### 管理员API (需Admin Token)
- `GET /api/admin/stats` - 统计数据
- `GET /api/admin/invites` - 邀请码列表
- `POST /api/admin/invites` - 创建邀请码
- `DELETE /api/admin/invites/:id` - 禁用邀请码
- `GET /api/admin/users` - 用户列表
- `GET /api/admin/users/:id` - 用户详情
- `DELETE /api/admin/users/:id` - 禁用用户
- `POST /api/admin/backup/all` - 全量备份
- `GET /api/admin/backup/all` - 下载全量备份
